# Аудит качества кода и UX — ЛК «Творцы РФ 2026»

_Дата: 2026-06-15. Метод: multi-agent аудит (11 измерений) с adversarial-проверкой каждой находки по фактическому коду. Подтверждено 56 из 82 находок (26 отсеяны как ложные/придирки/намеренные особенности прототипа)._

**Севернити:** 1 high · 18 medium · 37 low. **Срез:** 35 UX · 21 code-quality.

> Напоминание: это намеренный прототип (mock-auth/upload, демо-данные, захардкоженные даты). Такие вещи помечены как **прод-ловушка** — сейчас не баг, но мина при переносе на боевой `applications.tvortsy.online`.

---

## Резюме

Код прототипа в целом аккуратный и следует собственным конвенциям. Главные системные дыры — две: **доступность** (целый класс проблем — нигде нет видимого фокуса, у форм нет программной связи лейбл↔поле, модалки без фокус-трапа) и **набор логических краёв в доменных правилах** (доля 35+ обходится, synth-кап обходится при мультизагрузке, файл-ошибка не блокирует подачу). Отдельный пласт — **прод-ловушки**: незащищённая гидрация localStorage, заявки без владельца, отсутствие состояния «неверный код». Ни одного критического бага, ломающего прототип прямо сейчас, нет — но есть один high (фокус) и 18 medium, которые стоит закрыть до показа/интеграции.

**Чинить в первую очередь:** см. раздел [«Что в первую очередь»](#что-в-первую-очередь) в конце.

## Качество кода

### Корректность логики

- **[medium] Доля 35+ обходится приглашениями-пустышками** — `src/state/store.jsx:78-82`. `teamSeniorStat` числитель считает только реальных старших, а знаменатель `members.length` включает `invited` (возраст неизвестен) и `declined` (вышел). Команда из 4 реальных с 1 старшим: `floor(4·0.2)=0` → подать нельзя; добавил один `invited` без ответа → `floor(5·0.2)=1` → правило обойдено. → Считать долю по составу `tag in 'in'|'confirmed'`.
  - Связанный **[medium UX]** краевой текст «**X из 0 допустимых**» при малых командах (`max=floor(N·0.2)=0`) читается как баг системы (`ApplyForm.jsx:171,403`). → Для `max===0` дать человекочитаемую подсказку, а не «1 из 0».
- **[medium] Synth-кап суммарного размера обходится при мультизагрузке** — `src/screens/form/ApplyForm.jsx:110-117`. В цикле `addFiles` каждому файлу передаётся один и тот же старый снимок `app.files` (dispatch асинхронен), поэтому два файла по 300 МБ оба проходят, а сумма 600 > 500 МБ не ловится. → Аккумулировать нарастающим итогом `acc`.
- **[medium] Файл в состоянии `error` (неверный формат) не блокирует подачу** — `src/state/store.jsx:148-156`. `computeTodos` не покрывает одно из шести состояний файла: при наличии валидного `done` + `error` форма считает раздел готовым и разрешает подачу. Тихий провал валидации. → Добавить ветку `error` в `computeTodos`.
- **[medium] Сравнение email чувствительно к регистру** — `src/state/store.jsx:205-221`. `occupiesSlot`/`takenNominations`/`respond-invite`/`leave-team` сравнивают email точным `===` без нормализации. `K.Dmitriev@…` ≠ `k.dmitriev@…` → слот не резервируется, правило «2 на всё» подтекает. → Нормализовать `trim().toLowerCase()` при вводе и хранении.
- **[low] Асимметрия отклонения по пересланной ссылке** — `src/state/store.jsx:454-473`. Принятие незнакомца (`!known && 'confirmed'`) добавляет его в состав, а отклонение (`'declined'`) — no-op, `me` остаётся `undefined`, экран отказа может не показаться. → Обрабатывать оба пути симметрично.

### Прод-ловушки (mock сейчас — мина при интеграции)

- **[medium] Гидрация localStorage — shallow merge без валидации** — `src/state/store.jsx:658-665`. `{...initialState, ...JSON.parse(raw)}`: битый `apps` (не массив) → `.some/.filter is not a function` → белый экран, **error boundary в проекте нет**, самовосстановления нет. → Нормализовать/валидировать форму после parse; завести версию схемы вместо «молчаливого bump LS_KEY».
- **[medium] Заявки не имеют владельца** — `src/App.jsx:63-65`. `RequireActive` проверяет только stage; «своё» определяется вычитанием `own = apps.filter(a => !myMemberIn(a))`. В проде (общий стор с сервера) `/view`, `/apply`, `/success` для чужого id раскроют/дадут редактировать чужую заявку. → При интеграции добавить проверку владельца; минимум — TODO в гарде.
- **[medium] Неверный код подтверждения не имеет состояния ошибки вообще** — `src/screens/auth/Confirm.jsx:33-39`. Любые 4 цифры считаются верными; пути для ошибки в дизайне нет. Боевой API проглотит неверный код без обратной связи. → Зарезервировать состояние ошибки уже в прототипе (демо-правило, напр. `0000`=ошибка).
- **[low] `login`: guest → сразу `active`, минуя confirm/onboarding** — `src/state/store.jsx:385-391`. `keep` не содержит `guest`, фолбэк = `active`. Пользователь попадает в кабинет с пустым профилем. На боевом JWT «неизвестная стадия → active» недопустимо. → Стадию задаёт сервер; отметить в HANDOFF.
- **[low] Формат номера заявки ломается после 999** — `src/state/store.jsx:401`. `ТВ-2026-0${appSeq}`: на `appSeq=1000` будет 5 цифр; плюс дубль сид-номера `0847`. → `padStart(4,'0')`, выровнять сиды.
- **[low] Константы порога 35+ не источник истины** — `SENIOR_AGE=35` экспортируется, но `dobVerdict` зашивает `> 35` литералом (`store.jsx:67`); «20%» захардкожено текстом в 3 местах (`ApplyForm.jsx:171,404`, `Cabinet.jsx:188`) мимо `SENIOR_SHARE`. Правка константы соврёт пользователю / не изменит поведение. → Вывести проценты из `SENIOR_SHARE`, использовать `SENIOR_AGE` в `dobVerdict` (или удалить).

## Пользовательский опыт (UX)

### Доступность (a11y) — системный пробел

- **[HIGH] Нет видимого фокуса во всём проекте** — `src/styles/tokens.css:89`. Глобальный ресет снимает рамку, `:focus-visible` — 0 совпадений по репозиторию. Пользователь клавиатуры физически не видит, где он. WCAG 2.4.7. → Один глобальный блок `:where(button,a,[role=button],[tabindex]):focus-visible { outline: 2px solid var(--accent); outline-offset: 2px }`.
- **[medium] Лейблы не связаны с инпутами** — `src/components/ui.jsx:142-168`. Подпись — `<span>`, у `<input>` нет `id`/`htmlFor`/`aria-label`; `PasswordInput` вообще без связанного лейбла. Скринридер не зачитывает имя поля. Затрагивает все формы. → `useId()` + `<label htmlFor>`.
- **[medium] Нет `aria-invalid`/`aria-describedby`** — `ui.jsx:158-172`. Ошибка передаётся только цветом рамки; текст ошибки не связан с полем и не озвучивается. → `aria-invalid`, `aria-describedby`, `role="alert"`.
- **[medium] Модалки без `role="dialog"`/`aria-modal`/фокус-трапа/Esc** — `ui.jsx:306-310`. Деструктивные подтверждения (отзыв, удаление, выход из команды, смена пароля) — фокус «теряется» за модалкой, Esc не закрывает. → role/aria-modal, перенос и возврат фокуса, Esc, фокус-трап.
- **[medium] `Check` без `role="checkbox"`/`aria-checked`** — `ui.jsx:208-213`. Это **обязательные юридические согласия** (ПДн, Положение). Скринридер не сообщает, отмечено ли. → `role="checkbox"` + `aria-checked` или нативный input.
- **[medium] `Chips` без семантики `radiogroup`/`group`** — `ui.jsx:185-205`. Выбор («Одному»/«С командой», направления синтеза) не объявляется как активный, группа не связана с подписью. → `fieldset`/`legend` или `role="radiogroup"` + `aria-checked`.
- **[medium] Тосты не объявляются скринридером** — `src/state/store.jsx:712-714`. Контейнер без `aria-live`/`role=status`; критичная обратная связь («Приглашение отправлено», «У тебя уже 2 заявки») проходит мимо незрячих. WCAG 4.1.3. → `role="status"`, `aria-live="polite"`.
- **[low] `SynthCard`: вложенные кнопки внутри `role=button`** — `NominationCards.jsx:46-54`. Невалидная вложенность интерактивных элементов сбивает обход с клавиатуры. → Кликабельна только шапка; направления вынести из контейнера.
- **[low] Декоративные SVG без `aria-hidden`** — `ui.jsx:104-113` (`EyeIcon`/`StatusIcon`/иконки `FileRow`). В `Nav.jsx` сделано правильно — выровнять.
- **[low] Низкий контраст плейсхолдеров** — `forms.css:16`. `#9AA6B0` на белом ≈ 2.3:1 < 4.5:1. Плейсхолдер несёт пример формата. → Затемнить до ~`#646A6E`.

## Что в первую очередь

1. **`:focus-visible` глобально** (#1, HIGH) — одна строка CSS закрывает всю клавиатурную доступность.
2. **a11y-пакет в `ui.jsx`** (#9,#10,#11,#12,#13 + тосты #18) — лейблы, `aria-invalid`, Modal-трап, Check, Chips, `aria-live`. Почти всё в одном-двух файлах, эффект на весь продукт.
3. **`teamSeniorStat`: знаменатель по реальному составу** (#4) — обходимое доменное правило 35+.
4. **`computeTodos` учитывает файл-`error`** (#7) — закрыть тихий провал валидации подачи.
5. **Synth-кап суммарного размера при мультизагрузке** (#3).
6. **Защита гидрации localStorage + error boundary** (#5) и **гард auth-экранов для active** (#8).
7. **`copyInviteLink` — тост по факту** (#46) и **тач-цели `.mlink`** (#19).
8. **Прод-ловушки в HANDOFF** (#2 владелец заявки, #14 состояние ошибки кода, #31 login→active) — чтобы не уехали в боевой двойник незамеченными.

## Покрытие и оговорки аудита

- Синтез и критик полноты, а также 3 проверки измерения «адаптив» (`ApplyForm`, `ApplyView`, `index.html`) **не завершились** из-за лимита сессии — отчёт сведён вручную. **Адаптив формы заявки покрыт частично**, completeness-критик не отработал.
- Из 82 находок 26 отсеяны adversarial-проверкой (ложные/придирки/намеренные особенности прототипа) — в отчёт не попали.
- Прототипные допущения (mock-auth/upload, демо-строки, хардкод дат) не репортились как баги — только там, где ломают UX уже сейчас или создают мину при переносе в прод.
